## 컬럼 정의

- **슬러그**: 레포 식별자 (URL-safe)
- **원격**: 레포 원격 URL (GitLab 등) — *머신 독립 식별자*
- **역할**: 레포 한 줄 요약
- **도메인**: 담당 업무 도메인 (콤마 구분, 멀티)
- **의존**: 의존하는 다른 레포 슬러그 (콤마 구분, 멀티)

> **로컬 경로 해석**: 절대 경로는 이 표에 없음. 런타임(ROUTING §2 영향 분석·STEP 4 위탁)은 머신별 로컬 경로를 `.env`의 `{WORKSPACE_ROOT}` + 슬러그로 해석하거나, 없으면 원격에서 clone. (시크릿·머신별 값은 `.env`, 추적 안 됨.)

## 레포 프로파일 (조건부 — CI·릴리스·배포·자격증명·디자인 표면)

> **선택·조건부 보강.** 핵심 인벤토리(위 `{REPO_TABLE}`)와 분리된 *별도 블록* — 표를 비대화하지 않는다. 각 레포가 *실제로 보유한* 항목만 기록하고, **없으면 행 자체를 생략하거나 "없음"** 으로 둔다 (범용 툴 전제 — CI·릴리스·디자인 표면이 없는 레포도 정상).
> 작성 주체는 **레포 단위 셋업 에이전트(REPO-SETTER)** — 레포 인터뷰·자동 탐지로 채운다(원천: `agents/REPO-SETTER.md`). 핵심 표 행(SETTER·REPO-CREATOR)과 *대응 슬러그로 연결*된다.
> 도구·기술 명칭(CI 시스템·레지스트리·버전 도구·디자인 툴 등)은 모두 *예시(e.g.)* — 프로젝트·OS에 맞는 동치 수단으로 대체.
> **시크릿 경계**: 자격증명은 *이름·스코프·역할만* 기록한다. **값은 절대 적지 않는다** — 값은 머신별 `.env`(gitignore)에만 (POLICY-TRACKING / 자격증명 경계). 본 프로파일은 추적되는 공유 문서다.
> 연계: CI 게이트·릴리스·생산자/소비자·자격증명 사전조건은 `specs/CICD-RELEASE-ADAPTER.md`(**POLICY-RELEASE**) / 배포 타깃·전략·환경 티어·prod 트리거·배포 자격증명은 `specs/DEPLOY-ADAPTER.md`(**POLICY-DEPLOY**) + `agents/DEPLOY-AGENT.md`(배포 실행 시 소비) + prod 트리거 사람 게이트는 DECISIONS **A-4** / 디자인 표면의 정규 원천·동기화 방향은 `specs/SYSTEM-WORKFLOW.md` §3(디자인↔코드 동기화 방향) + DECISIONS **A-2** / 생산자→소비자 채택 게이트는 DECISIONS **A-5** / 자격증명 스코프 진단은 ERROR-POLICY **EX-13**.

{REPO_PROFILES}

> **프로파일 항목 정의** (각 레포 서브섹션이 채우는 필드 — 해당 없으면 생략/"없음"):
>
> - **CI/CD**: CI 시스템 보유 여부 + 머지를 *차단(blocking)* 하는 필수 게이트(예: 빌드·테스트·린트) + *사람 승인 게이트*(예: 시각회귀 리뷰 등 자동 통과 불가 게이트)가 있으면 그 성격. — POLICY-RELEASE §2·§3.
> - **릴리스/퍼블리시**: 산출물을 퍼블리시하는가 + 릴리스 방식(버전 범프·태그·CI 퍼블리시 등 레포 컨벤션) + 대상 레지스트리·엔드포인트(검증용 좌표 — POLICY-VERIFY §4[5] 존재 질의 대상). — POLICY-RELEASE §4·§8.
> - **배포(deploy)** *(조건부 — 실행 환경에 배포하는 레포만; 라이브러리·산출물만 퍼블리시하는 레포는 생략)*: 배포 **타깃**(어디로 — 예: 임의의 실행 환경·플랫폼) + **방식·전략**(예: rolling·blue-green·canary·recreate — 레포가 포착한 방식) + **환경 티어**(예: dev·staging·prod — 낮은→높은 승격 위상) + **prod 트리거 주체**(누가 prod 배포를 승인·트리거하는가 — A-4 사람 게이트 입력) + **배포 자격증명 이름·스코프**(예: 환경 접근 키·배포 권한 롤 — **이름·스코프만, 값 금지**, `.env`에만). 이 필드는 `DEPLOY-AGENT`가 배포 시점에 소비한다. — POLICY-DEPLOY §2·§3·§4·§9 / DEPLOY-AGENT D0 / DECISIONS A-4.
> - **자격증명 사전조건**: CI·릴리스·자동화 작업이 요구하는 자격증명의 **이름·스코프·역할만**(예: 상태 쓰기 스코프·퍼블리시 쓰기 스코프·레지스트리 역할 등). **값 금지** — `.env`에만. — POLICY-RELEASE §6 / EX-13 / 자격증명 경계.
> - **디자인 표면**: 이 레포에 대응하는 디자인 원천이 있으면 그 위치(임의의 디자인 툴·파일·보드 *또는* 별도 스펙 산출물) + **정규(canonical) 원천**(코드 vs 디자인 등) + **동기화 방향**(무엇이 이끌고 무엇이 따라가는지). 방향 결정은 사이클마다 A-2(항상 컨펌)로 재확인 — 여기엔 *현재 합의된 기본*만 기록. — SYSTEM-WORKFLOW §3 / DECISIONS A-2.
> - **producer/consumer 관계**: 이 레포가 다른 레포들이 소비하는 *공유 생산자*인지, 아니면 다른 레포의 퍼블리시 산출물을 쓰는 *소비자*인지(상대 슬러그 명시 — A-5 채택 게이트 입력). — POLICY-RELEASE §5 / DECISIONS A-5.

> **`{REPO_PROFILES}` 형식** — 프로파일을 *보유한* 레포마다 하나의 서브섹션(슬러그 헤딩 + 보유 필드만). 어떤 레포도 프로파일이 없으면 블록 전체를 `_(해당 프로파일 없음)_` 한 줄로 둔다. 서브섹션 형태(예시):
>
> ```markdown
> ### {repo-slug}
>
> - **CI/CD**: {CI 시스템 — 예: 파이프라인 정의 보유} / 차단 게이트: {예: build·test·lint} / 사람 승인 게이트: {예: 시각회귀 리뷰 — 없으면 생략}
> - **릴리스/퍼블리시**: {퍼블리시 여부·방식} / 레지스트리·엔드포인트: {검증용 좌표 — 없으면 "없음"}
> - **배포**: 타깃: {어디로 — 예: 실행 환경·플랫폼} / 방식·전략: {예: rolling·blue-green·canary·recreate} / 환경 티어: {예: dev·staging·prod} / prod 트리거 주체: {누가 prod 승인·트리거 — A-4} / 배포 자격증명: {이름·스코프만 — 예: 환경 접근 키·배포 권한 롤. 값 금지(`.env`)}  ← *배포하지 않는 레포는 행 생략*
> - **자격증명 사전조건**: {이름·스코프·역할만 — 예: 퍼블리시 쓰기 스코프, 상태 쓰기 스코프. 값 금지(`.env`)}
> - **디자인 표면**: {위치 — 디자인 툴·파일·보드 또는 스펙 산출물} / 정규: {코드|디자인|…} / 동기화 방향: {정규→종속}
> - **producer/consumer**: {producer: 소비 레포 슬러그[] | consumer: 생산 레포 슬러그[]}
> ```
>
> 보유하지 않는 필드는 *행 자체를 생략* (빈 `{...}` 자리 표시자 잔존 금지).
