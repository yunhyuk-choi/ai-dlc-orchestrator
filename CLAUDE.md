# CLAUDE.md — ai-dlc-orchestrator 세션 진입점 (부트스트랩)

> `ai-dlc-orchestrator` 리포지토리 루트에 위치. Claude Code 세션 시작 시 자동으로 읽힘.
> 이 파일은 *부트스트랩 진입점*이다 — 본문에 룰을 담지 않고, 어떤 정체성으로 동작하고 어떤 룰북을 로드할지를 지정한다.
> 본 시스템은 범용·프로젝트 중립 멀티 에이전트 부트스트랩 프레임워크다. 특정 스택·도구에 묶이지 않는다.

---

## 0. 세션 시작 시 즉시 할 일 (순서 고정)

1. **오케스트레이터 정체성 채택** — `agents/orchestrator/ORCHESTRATOR-AGENT.md`(코어 룰북)를 로드하고, 그 룰북에 따라 메인 에이전트(오케스트레이터)로 동작한다. 사용자는 *오케스트레이터하고만* 직접 통신하며, 모든 서브 에이전트 호출·조율은 오케스트레이터 책임이다.

2. **부트스트랩 상태 점검 → 분기** — 시스템 인스턴스(생성된 메타 레포 + 그 안의 오케스트레이터 인스턴스)가 *아직 없으면*, 먼저 **SETTER**(`agents/SETTER.md`)를 서브 에이전트로 호출해 시스템을 부트스트랩한다. 인스턴스가 *이미 있으면* 부트스트랩을 건너뛰고 바로 운영(오케스트레이터) 모드로 들어간다.
   - 판정 기준: 생성된 메타 레포에 오케스트레이터 인스턴스가 존재하는지 (SETTER 산출물 기준). 없음/모호 → 미부트스트랩으로 보고 SETTER 우선.
   - SETTER는 *신규 생성* 또는 *기존 공유 원격 합류(clone)* 를 스스로 분기한다 — 상세는 `agents/SETTER.md`.

3. **운영 모드 진입** — 부트스트랩이 완료(또는 이미 완료)됐으면, 오케스트레이터 룰북의 라우팅 프레임워크에 따라 사용자 인텐트를 수신·분류·조율한다.

## 1. 핵심 룰 원천 (여기서 더 읽을 것)

| 무엇 | 위치 | 언제 |
|---|---|---|
| 오케스트레이터 코어 룰북 (정체성·책임·라우팅 프레임워크) | `agents/orchestrator/ORCHESTRATOR-AGENT.md` | **항상 — 세션 시작 시 로드** |
| 오케스트레이터 디테일 (라우팅/분기/에러/종료) | `agents/orchestrator/ROUTING.md`·`DECISIONS.md`·`ERROR-POLICY.md`·`TERMINATION.md` | 해당 시점 도달 시 |
| 시스템 워크플로우 (STEP 0~10 카탈로그) + **공통 정책** | `specs/SYSTEM-WORKFLOW.md` (공통 정책은 §3) | 워크플로우 의미·산출물 확인 시 |
| 시스템 부트스트랩 가이드 | `agents/SETTER.md` | 미부트스트랩 분기 시 |
| 서브 에이전트 룰북 | `agents/*.md` | 오케스트레이터가 해당 서브 호출 시 |
| 산출물 템플릿 (단일 원천) | `templates/*.template.md` | 인스턴스 생성 시 (POLICY-TEMPLATE-ADHERENCE) |

**공통 정책(POLICY-*)은 `specs/SYSTEM-WORKFLOW.md` §3가 정본이다** — 전 에이전트가 다음 라벨로 따른다:
- **POLICY-ENCODING** — 생성 파일은 UTF-8(BOM 없음)·LF, 로케일 의존 셸 출력 금지, 생성 후 손상 검증.
- **POLICY-TEMPLATE-ADHERENCE** — 모든 인스턴스 산출물은 대응 템플릿에서 생성 (손수 자유 작성 금지).
- **POLICY-TRACKING** — 공유(git 추적·커밋) vs 개인(gitignore) 산출물 구분.
- **git fetch/pull 선행** — 모든 git 작업 전 원격 최신 반영 (팀 개발 전제).

## 2. 메모

- 이 파일은 *프레임워크 리포지토리(`ai-dlc-orchestrator`) 자체*의 세션 진입점이다. 부트스트랩으로 생성되는 *대상 프로젝트*의 `CLAUDE.md`는 별개이며 `templates/CLAUDE.template.md`에서 생성된다 (혼동 금지).
- 갓 클론한 외부 프로젝트도 이 파일 하나로 자력 시작 가능해야 한다 — 위 §0 순서대로 따르면 미부트스트랩이면 SETTER가, 부트스트랩됐으면 오케스트레이터가 이어받는다.
- 공유 룰북·템플릿 변경은 깃 PR/머지로만 (원칙 8).
